Day 8: Selection Structure – Ternary & Switch Case
🧪 Ternary Operator:
- Syntax: condition ? expression1 : expression2;
- Used for short conditional assignments or decisions.
- Example: int max = (a > b) ? a : b;

🎛️ Switch Case:
- Used when we have multiple conditions based on a single variable.
- Syntax: 
switch(variable) {
        case value1:
            // code
            break;
        case value2:
            // code
            break;
        default:
            // default code
    }
- Ideal for menu-based programs or decision trees.

💡 Tips:
  • Ternary is best for simple conditions.
  • Switch is better for checking multiple fixed values.
  • Always include break in cases to avoid fall-through.
  • default is optional but useful as a fallback.

Day 24: Pointers, Array of Pointers & Chain of Pointers
🔹 Pointers:
- Definition: A pointer is a variable that stores the memory address of another variable.
- Syntax: datatype *pointerName;
- Example: 
int a = 5;
    int *p = &a; // p stores the address of a
- Important: Printing a pointer directly shows the memory address. Use *p to access the value stored at that address.

🔸 Array of Pointers:
- Definition: An array that stores addresses instead of values.
- Example: 
int a = 10, b = 20, c = 30;
    int *arr[3] = {&a, &b, &c};
- Useful for handling arrays of strings, dynamic arrays, or when working with functions like pointers to functions.

🔗 Chain of Pointers (Pointer to Pointer):
- Definition: A pointer that stores the address of another pointer.
- Example: 
int x = 50;
    int *p = &x;
    int **q = &p; // q stores address of p
- Useful in advanced data structures like dynamic 2D arrays and when passing pointers by reference.

🧠 Quick Tip:
- Remember: Each * represents a level of indirection.
- Visualize the memory links to understand pointer chains better!

Day 25: Reference Variables & Argument Passing
📌 Reference Variables:
- Definition: A reference is an alias for another variable (it shares the same memory address).
- Syntax: datatype &referenceName = originalVariable;
- Example: 
int x = 5;
    int &y = x; // y is now another name for x
- Changes through y will directly affect x.

📌 Argument Passing:
Understanding how data is passed to functions.

Types of Argument Passing:
  • Passing a Value (Primitive): Pass simple variables. 
    void show(int a) { cout << a; }
    show(x);
  • Passing an Array: Arrays automatically pass their address. 
    void display(int arr[], int n) { /* code */ }
    display(arr, 5);
  • Passing a Pointer: Pass the memory address. 
    void update(int *p) { *p = 10; }
    update(&x);
  • Passing an Address: Same as passing a pointer. 
    void increment(int *ptr) { (*ptr)++; }
    increment(&a);
📂 Function Calling Types:

1. Call by Value:
- Copies the actual value.
- Changes inside the function do not affect the original variable.
void change(int a) { a = 10; }

2. Call by Reference:
- Uses reference variables.
- Changes inside the function affect the original variable.
void change(int &a) { a = 10; }

3. Call by Const Reference:
- Used to pass large objects without copying, while preventing changes.
void print(const int &a) { cout << a; }

🧠 Tips:
- Call by value is safer for small variables.
- Call by reference is best for memory efficiency.
- Const reference is ideal when you want safety plus performance.

Day 26: 🧩 Object-Oriented Programming & Key Concepts
📚 What is Object-Oriented Programming (OOP)?
- OOP is a programming paradigm based on the concept of objects that contain both data (attributes) and functions (methods).

⚙️ Principles of OOP:
  • Encapsulation – Binding data and methods together in a class.
  • Abstraction – Hiding unnecessary details and showing only essential features.
  • Inheritance – Reusing features of an existing class in a new class.
  • Polymorphism – Performing a single action in different ways.

🏗️ Class & Object:
- Class: A blueprint or template for creating objects.
- Object: An instance of a class.

🔓 Access Modifiers:
- Public: Accessible from anywhere.
- Private: Accessible only within the class.

🔐 Data Encapsulation
Definition: Wrapping data and methods that operate on the data into a single unit (class) and restricting direct access.

Setter & Getter: Special methods to safely modify (set) or access (get) private data.

Example: 
class Student {
    private:
        int age;
    public:
        void setAge(int a) { age = a; }
        int getAge() { return age; }
    };
📌 this Keyword
Definition: A pointer that points to the current object.
Purpose: Used to resolve conflicts between class attributes and parameters or to return the current object.

Example: 
class Student {
    private:
        int age;
    public:
        void setAge(int age) { this->age = age; }
        Student* setData(int age) {
            this->age = age;
            return this;
        }
    };
  • this helps when parameter names are same as attribute names.
  • You can return this for method chaining.
🔒 Private Attributes
Definition: Attributes declared as private are not directly accessible outside the class.

Use Cases:
  • Protect sensitive data.
  • Force controlled access using setters/getters.
  • Helps maintain data integrity.

Example: 
class BankAccount {
    private:
        double balance;
    public:
        void deposit(double amount) { balance += amount; }
        double getBalance() { return balance; }
    };

Day 27: 🧩 Array of Objects, Static Members & Scope Resolution Operator
1) Array of Objects
- An array that holds multiple objects of the same class.
- Syntax example:
ClassName arr[size];
- Use setters & getters to modify/access object properties.

Example:

class Student {
  int id;
  string name;
public:
  void setData(int i, string n) { id = i; name = n; }
  void display() { cout << id << " " << name << endl; }
};

Student arr[3];  // array of 3 Student objects
2) Scope Resolution Operator (::)
- Used to access global variables when shadowed by local variables.
- Used to access static members of a class.
- Syntax example:

::globalVar;            // Access global variable
ClassName::staticVar;   // Access static data member
ClassName::staticFunc(); // Call static member function
3) static Keyword
- Used to declare static data members and static member functions.
- Static data members are shared among all objects of the class.
- Static member functions can access only static data members.

Use Cases:
- Count total objects created.
- Share common properties without duplicating in each object.

Example:

class Counter {
  static int count;  // static data member
public:
  Counter() { count++; }
  static int getCount() { return count; }
};

int Counter::count = 0;  // definition outside class

Counter c1, c2;
cout << Counter::getCount();  // Output: 2

Day 28: 🧩 Constructor, Types & Destructor
1) Constructor
- A special function that automatically runs when an object is created.
- No return type (not even void) and has the same name as the class.

🔹 Types of Constructor:
  • Default Constructor: No parameters.
    Student() { cout << "Default Constructor"; }

- Special function automatically called when an object is destroyed.
- Used for cleanup tasks like freeing memory.
- Same name as class but prefixed with ~.

        class Student {

        Student() { cout << "Default Constructor" << endl; }

        };

Day 29: 🧬 Inheritance & Ambiguity Handling
1) Inheritance
- Inheritance allows a class (child/derived class) to acquire properties and behaviors from another class (parent/base class).

🔹 Key Concepts:
- Base Class: Parent class
- Derived Class: Child class that inherits

🔹 Syntax:
class Derived : public Base { 
        // body
        };

🔹 Types of Inheritance:
  • Single Inheritance: One base → one derived class
  • Multilevel Inheritance: Derived from a derived class
  • Multiple Inheritance: Derived from multiple base classes
  • Hierarchical Inheritance: Multiple derived classes from a single base class
  • Hybrid Inheritance: Combination of two or more inheritance types
2) Ambiguity in Hybrid Inheritance
- Occurs when a derived class has multiple paths to access the same member from the base classes.
- Leads to confusion: compiler doesn't know which path to follow.

🔸 Example of Ambiguity:

        class A {
        public:
            void show() { cout << "Class A" << endl; }
        };

        class B : public A {};
        class C : public A {};
        class D : public B, public C {}; // Ambiguity
Solution: Membership Label Operator
- Use class name with scope resolution operator to tell the compiler which class member to call.

🔧 Solution Example:
D obj;
        obj.B::show();  // Calls show() from class B
        obj.C::show();  // Calls show() from class C

Day 30: 🔄 Polymorphism & Method Overloading/Overriding
🧠 Polymorphism:
- Polymorphism means "many forms". It allows methods to behave differently based on context.

📚 Types of Polymorphism:
- Compile-time (Static): Method Overloading
- Run-time (Dynamic): Method Overriding & Virtual Functions

⚙️ Method Overloading:
- Same method name, different parameters (type or number).
- Purpose: Increase readability and flexibility.
- Binding: Resolved at compile time.
- Example: 
void print(int x);
    void print(double y);
    void print(int x, int y);

🔄 Method Overriding:
- Derived class provides specific implementation of a base class method.
- Purpose: Runtime polymorphism, dynamic behavior.
- Binding: Resolved at runtime via virtual functions.
- Visibility: Base class method should be virtual.
- Example: 
class Base {
    public:
    virtual void show();
    };
    class Derived : public Base {
    public:
    void show() override;
    };

Day 31: 🖥️ Abstraction & Access Modifiers
1) Abstraction
  • Definition: Hiding internal implementation details and showing only necessary features.
  • Key Concepts:
    • Abstract Class: A class that cannot be instantiated and contains at least one pure virtual function.
    • Pure Virtual Function: A function declared in a base class that must be overridden in derived classes. Syntax: virtual void show() = 0;
  • Example: Abstract base class Shape with pure virtual method draw().
2) Access Modifiers in Inheritance
  • Public: Derived class inherits public and protected members as-is.
  • Protected: Derived class treats all inherited public members as protected.
  • Private: All inherited members become private in derived class.
3) Visibility Modifiers (Access Specifiers)
  • Public: Accessible everywhere.
  • Protected: Accessible within the class and its subclasses.
  • Private: Accessible only within the class itself.
  • Note: Although "visibility" and "access" are sometimes used interchangeably, they refer to the same concept in C++.

Day 32: 🧠 Static vs. Dynamic Memory Allocation
1) Static Memory Allocation (SMA)
  • Definition: Memory is allocated at compile time.
  • Example: int x = 10; — memory is reserved before program runs.
  • Scope: Limited to the block in which it is defined.
2) Memory Structure of RAM
  • Code Section: Stores compiled instructions.
  • Global/Static Section: Stores global and static variables.
  • Stack: Stores function calls, local variables. Grows downwards.
  • Heap: Used for dynamic memory. Grows upwards.
3) Dynamic Memory Allocation (DMA)
  • Definition: Memory allocated during runtime using new and released using delete.
  • Need: When we don't know how much memory is required beforehand (e.g., user input size).
  • Example: int* ptr = new int; → allocates memory in heap.
  • Manual Deallocation: delete ptr;
Stack vs Heap (Detailed)
  • Stack:
    • Automatically managed
    • Fast but limited size
    • Used for function calls, local vars
  • Heap:
    • Manually managed
    • Slower but more flexible
    • Used for objects, arrays created via new

Day 33: ⚙️ DMA with Procedural vs. OOP
1) DMA in Procedural Style
  • Uses new and delete with basic data types or structures.
  • No encapsulation or object behavior.
  • Example: 
    int* ptr = new int(5); 
        cout << *ptr; 
        delete ptr;
2) DMA in OOP Style
  • Objects created with new trigger the constructor.
  • delete calls the destructor automatically.
  • Encapsulates both data and behavior.
  • Example: 
    class Student {
        public:
            Student() { cout << "Constructor called\n"; }
            ~Student() { cout << "Destructor called\n"; }
        };

        Student* s = new Student(); 
        delete s;

Day 34: 🧩 Introduction to Templates
🔹 What is a Template?
  • A C++ template allows writing generic and reusable code.
  • Instead of writing the same logic for different data types, we use templates.
  • It gets replaced with actual data types during compilation.
🔸 Types of Templates
1. Function Template
  • Generic function that works with different data types.
  • Example: 
    template <typename T>
        T add(T a, T b) {
        return a + b;
        }

        // Usage:
        cout << add<int>(3, 4);   // 7
        cout << add<float>(3.5, 2.5); // 6.0
2. Class Template
  • Used to create a class that can handle multiple data types.
  • Example: 
    template <class T>
        class Box {
        T value;
        public:
        void set(T v) { value = v; }
        T get() { return value; }
        };

        // Usage:
        Box<int> intBox;
        intBox.set(100);
        cout << intBox.get();

Tip: Templates increase code reusability and reduce redundancy in logic.

Day 35: 🧺 Vectors and Array Comparison
📦 What is a Vector?
  • Vectors are dynamic arrays in C++ (part of STL).
  • They automatically manage memory and can grow/shrink at runtime.
🧾 Syntax & Example
#include <vector>
        #include <iostream>
        using namespace std;

        vector<int> v = {1, 2, 3};
        v.push_back(4);
🔧 Vector Methods
  • push_back(val) – adds element at end
  • pop_back() – removes last element
  • insert(it, val) – inserts at position
  • erase(it) – deletes from position
  • clear() – removes all elements
  • size() – returns current size
  • empty() – checks if vector is empty
  • at(i) – returns element at index i
  • front() / back() – first / last elements
  • begin() / end() – iterators to start / end
🔄 Vector vs Array
FeatureArrayVector
SizeFixedDynamic
Memory ManagementManualAutomatic
STL SupportNoYes
Bounds CheckNoat() checks
Ease of UseBasicFlexible & rich

Tip: Always prefer vectors over arrays when working with dynamic or unknown sizes.

Day 37: 🔗 Linked List & Its Operations
📌 What is a Linked List?
  • A Linked List is a linear data structure where elements (nodes) are stored in memory using pointers.
  • Each node contains:
    • data: the actual value
    • next: pointer to the next node
🔧 Structure of a Node
struct Node {
    int data;
    Node* next;
};
✅ Advantages
  • Dynamic size (no fixed limit)
  • Efficient insertions/deletions
❌ Disadvantages
  • Sequential access only (no direct indexing)
  • Extra memory for pointers
🔁 Types of Linked List
  • Singly Linked List: One pointer to next node only
  • Doubly Linked List: Two pointers (next and previous)
  • Circular Linked List: Last node points back to the head
  • Circular Doubly Linked List: Combines both
⚙️ Operations in Linked List
📥 Insertion
void insertAtFront(Node*& head, int value) {
    Node* newNode = new Node{value, head};
    head = newNode;
}
🗑️ Deletion
void deleteFront(Node*& head) {
    if (head) {
        Node* temp = head;
        head = head->next;
        delete temp;
    }
}
🔍 Traversal
void printList(Node* head) {
    while (head) {
        cout << head->data << " ";
        head = head->next;
    }
}
🔎 Searching
bool search(Node* head, int key) {
    while (head) {
        if (head->data == key) return true;
        head = head->next;
    }
    return false;
}

Tip: Mastering node manipulation helps in understanding dynamic structures and problem-solving in interviews.

Day 38: 🔁 Creation of Singly Linked List

// ✅ 1) Singly Linked List:
- Definition:
  A singly linked list is a linear data structure where each element (node) points to the next.

- Node:
  A structure containing:
    - `data`: value stored in the node
    - `next`: pointer to the next node

- Structure:
  class Node {
      public:
          int data;
          Node* next;
  };

- Example:
  Node* first = new Node();
  first->data = 10;
  first->next = NULL;

- Step-by-step creation:
  • Create 1st node → assign data → next = NULL
  • Create 2nd node using separate pointer → link first->next = second
  • Create 3rd node → link second->next = third
  (and so on...)

- Node Representation:
  [data | next] → [data | next] → NULL

- Problem:
  ❗ Using separate pointer for each node becomes impractical for large numbers (e.g., 20+ nodes).
  ❗ Better solution: use loops or dynamic techniques (e.g., using tail pointers or classes with methods).

// 🚀 Summary:
- Self-referential class allows dynamic node creation.
- Demonstrates importance of proper memory management in linked lists.
- Sets the stage for implementing insertion/deletion using loops.
