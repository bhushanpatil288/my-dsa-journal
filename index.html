🧮 Types of Operators:
  • Arithmetic: +, -, *, /, %
  • Relational: ==, !=, >, <, >=, <=
  • Logical: &&, ||, !

🔢 Operator Precedence:
- Some operators are executed before others in an expression.
- Example: int x = 5 + 2 * 3; → Multiplication runs first.
- Use () to control order.
🔁 Type Conversion:
- Converting a value from one data type to another.

🔸 Implicit Conversion:
- Done automatically by the compiler.
- Example: int + float → int converts to float

🔹 Explicit Conversion (Type Casting):
- Done manually by programmer.
- Example: (int)3.14 → becomes 3
Day 7:🧠 Control Structures and 🔷 Flowchart in Programming
📍 Types of Control Structures:

🔹 Sequential Structure:
- Code executes line by line from top to bottom.

🔸 Selection Structure:
- Decision-making structures based on conditions.
  • if – Runs block if condition is true.
  • if-else – One block runs if true, another if false.
  • if-else-if Ladder – Multiple conditions checked in sequence.
  • Nested if – if inside another if.
  • Ternary – condition ? trueBlock : falseBlock;
  • switch-case – Selects block based on value (good alternative to long if-else chains).

🔁 Repetition Structure:
- Used for looping (covered later in detail).
📌 What is a Flowchart?
- A diagram that shows the sequence of steps in a program.
- Helps plan, debug, and explain logic clearly.

📐 Common Symbols:
  • Terminator – 🟢 Start/End (Oval)
  • Process – 🔲 Operation or action (Rectangle)
  • Decision – 🔷 Condition check (Diamond)
  • Input/Output – 🔺 Input/output operations (Parallelogram)
  • Arrow – ➡️ Shows the direction of flow

🛠️ Example Use:
- Before writing actual code, design a flowchart to get clarity.
- Especially helpful for beginners and complex logic.
Day 8: Selection Structure – Ternary & Switch Case
🧪 Ternary Operator:
- Syntax: condition ? expression1 : expression2;
- Used for short conditional assignments or decisions.
- Example: int max = (a > b) ? a : b;

🎛️ Switch Case:
- Used when we have multiple conditions based on a single variable.
- Syntax: 
switch(variable) {
        case value1:
            // code
            break;
        case value2:
            // code
            break;
        default:
            // default code
    }
- Ideal for menu-based programs or decision trees.

💡 Tips:
  • Ternary is best for simple conditions.
  • Switch is better for checking multiple fixed values.
  • Always include break in cases to avoid fall-through.
  • default is optional but useful as a fallback.

Day 24: Pointers, Array of Pointers & Chain of Pointers
🔹 Pointers:
- Definition: A pointer is a variable that stores the memory address of another variable.
- Syntax: datatype *pointerName;
- Example: 
int a = 5;
    int *p = &a; // p stores the address of a
- Important: Printing a pointer directly shows the memory address. Use *p to access the value stored at that address.

🔸 Array of Pointers:
- Definition: An array that stores addresses instead of values.
- Example: 
int a = 10, b = 20, c = 30;
    int *arr[3] = {&a, &b, &c};
- Useful for handling arrays of strings, dynamic arrays, or when working with functions like pointers to functions.

🔗 Chain of Pointers (Pointer to Pointer):
- Definition: A pointer that stores the address of another pointer.
- Example: 
int x = 50;
    int *p = &x;
    int **q = &p; // q stores address of p
- Useful in advanced data structures like dynamic 2D arrays and when passing pointers by reference.

🧠 Quick Tip:
- Remember: Each * represents a level of indirection.
- Visualize the memory links to understand pointer chains better!

Day 25: Reference Variables & Argument Passing
📌 Reference Variables:
- Definition: A reference is an alias for another variable (it shares the same memory address).
- Syntax: datatype &referenceName = originalVariable;
- Example: 
int x = 5;
    int &y = x; // y is now another name for x
- Changes through y will directly affect x.

📌 Argument Passing:
Understanding how data is passed to functions.

Types of Argument Passing:
  • Passing a Value (Primitive): Pass simple variables. 
    void show(int a) { cout << a; }
    show(x);
  • Passing an Array: Arrays automatically pass their address. 
    void display(int arr[], int n) { /* code */ }
    display(arr, 5);
  • Passing a Pointer: Pass the memory address. 
    void update(int *p) { *p = 10; }
    update(&x);
  • Passing an Address: Same as passing a pointer. 
    void increment(int *ptr) { (*ptr)++; }
    increment(&a);
📂 Function Calling Types:

1. Call by Value:
- Copies the actual value.
- Changes inside the function do not affect the original variable.
void change(int a) { a = 10; }

2. Call by Reference:
- Uses reference variables.
- Changes inside the function affect the original variable.
void change(int &a) { a = 10; }

3. Call by Const Reference:
- Used to pass large objects without copying, while preventing changes.
void print(const int &a) { cout << a; }

🧠 Tips:
- Call by value is safer for small variables.
- Call by reference is best for memory efficiency.
- Const reference is ideal when you want safety plus performance.

Day 26: 🧩 Object-Oriented Programming & Key Concepts
📚 What is Object-Oriented Programming (OOP)?
- OOP is a programming paradigm based on the concept of objects that contain both data (attributes) and functions (methods).

⚙️ Principles of OOP:
  • Encapsulation – Binding data and methods together in a class.
  • Abstraction – Hiding unnecessary details and showing only essential features.
  • Inheritance – Reusing features of an existing class in a new class.
  • Polymorphism – Performing a single action in different ways.

🏗️ Class & Object:
- Class: A blueprint or template for creating objects.
- Object: An instance of a class.

🔓 Access Modifiers:
- Public: Accessible from anywhere.
- Private: Accessible only within the class.

🔐 Data Encapsulation
Definition: Wrapping data and methods that operate on the data into a single unit (class) and restricting direct access.

Setter & Getter: Special methods to safely modify (set) or access (get) private data.

Example: 
class Student {
    private:
        int age;
    public:
        void setAge(int a) { age = a; }
        int getAge() { return age; }
    };
📌 this Keyword
Definition: A pointer that points to the current object.
Purpose: Used to resolve conflicts between class attributes and parameters or to return the current object.

Example: 
class Student {
    private:
        int age;
    public:
        void setAge(int age) { this->age = age; }
        Student* setData(int age) {
            this->age = age;
            return this;
        }
    };
  • this helps when parameter names are same as attribute names.
  • You can return this for method chaining.
🔒 Private Attributes
Definition: Attributes declared as private are not directly accessible outside the class.

Use Cases:
  • Protect sensitive data.
  • Force controlled access using setters/getters.
  • Helps maintain data integrity.

Example: 
class BankAccount {
    private:
        double balance;
    public:
        void deposit(double amount) { balance += amount; }
        double getBalance() { return balance; }
    };

Day 27: 🧩 Array of Objects, Static Members & Scope Resolution Operator
1) Array of Objects
- An array that holds multiple objects of the same class.
- Syntax example:
ClassName arr[size];
- Use setters & getters to modify/access object properties.

Example:

class Student {
  int id;
  string name;
public:
  void setData(int i, string n) { id = i; name = n; }
  void display() { cout << id << " " << name << endl; }
};

Student arr[3];  // array of 3 Student objects
2) Scope Resolution Operator (::)
- Used to access global variables when shadowed by local variables.
- Used to access static members of a class.
- Syntax example:

::globalVar;            // Access global variable
ClassName::staticVar;   // Access static data member
ClassName::staticFunc(); // Call static member function
3) static Keyword
- Used to declare static data members and static member functions.
- Static data members are shared among all objects of the class.
- Static member functions can access only static data members.

Use Cases:
- Count total objects created.
- Share common properties without duplicating in each object.

Example:

class Counter {
  static int count;  // static data member
public:
  Counter() { count++; }
  static int getCount() { return count; }
};

int Counter::count = 0;  // definition outside class

Counter c1, c2;
cout << Counter::getCount();  // Output: 2

Day 28: 🧩 Constructor, Types & Destructor
1) Constructor
- A special function that automatically runs when an object is created.
- No return type (not even void) and has the same name as the class.

🔹 Types of Constructor:
  • Default Constructor: No parameters.
    Student() { cout << "Default Constructor"; }
  • Parameterized Constructor: Accepts parameters.
    Student(int id) { this->id = id; }
  • Copy Constructor: Creates a copy of an existing object.
    Student(const Student &s) { this->id = s.id; }
2) Destructor
- Special function automatically called when an object is destroyed.
- Used for cleanup tasks like freeing memory.
- Same name as class but prefixed with ~.

~Student() { cout << "Destructor Called"; }

🔧 Example:

        class Student {
        int id;
        public:
        Student() { cout << "Default Constructor" << endl; }
        Student(int i) { id = i; cout << "Parameterized Constructor" << endl; }
        Student(const Student &s) { id = s.id; cout << "Copy Constructor" << endl; }
        ~Student() { cout << "Destructor Called" << endl; }
        };

Day 29: 🧬 Inheritance & Ambiguity Handling
1) Inheritance
- Inheritance allows a class (child/derived class) to acquire properties and behaviors from another class (parent/base class).

🔹 Key Concepts:
- Base Class: Parent class
- Derived Class: Child class that inherits

🔹 Syntax:
class Derived : public Base { 
        // body
        };

🔹 Types of Inheritance:
  • Single Inheritance: One base → one derived class
  • Multilevel Inheritance: Derived from a derived class
  • Multiple Inheritance: Derived from multiple base classes
  • Hierarchical Inheritance: Multiple derived classes from a single base class
  • Hybrid Inheritance: Combination of two or more inheritance types
2) Ambiguity in Hybrid Inheritance
- Occurs when a derived class has multiple paths to access the same member from the base classes.
- Leads to confusion: compiler doesn't know which path to follow.

🔸 Example of Ambiguity:

        class A {
        public:
            void show() { cout << "Class A" << endl; }
        };

        class B : public A {};
        class C : public A {};
        class D : public B, public C {}; // Ambiguity
Solution: Membership Label Operator
- Use class name with scope resolution operator to tell the compiler which class member to call.

🔧 Solution Example:
D obj;
        obj.B::show();  // Calls show() from class B
        obj.C::show();  // Calls show() from class C

Day 30: 🔄 Polymorphism & Method Overloading/Overriding
🧠 Polymorphism:
- Polymorphism means "many forms". It allows methods to behave differently based on context.

📚 Types of Polymorphism:
- Compile-time (Static): Method Overloading
- Run-time (Dynamic): Method Overriding & Virtual Functions

⚙️ Method Overloading:
- Same method name, different parameters (type or number).
- Purpose: Increase readability and flexibility.
- Binding: Resolved at compile time.
- Example: 
void print(int x);
    void print(double y);
    void print(int x, int y);

🔄 Method Overriding:
- Derived class provides specific implementation of a base class method.
- Purpose: Runtime polymorphism, dynamic behavior.
- Binding: Resolved at runtime via virtual functions.
- Visibility: Base class method should be virtual.
- Example: 
class Base {
    public:
    virtual void show();
    };
    class Derived : public Base {
    public:
    void show() override;
    };
